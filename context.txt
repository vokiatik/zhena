PROJECT OVERVIEW

The project is a local AI-powered analytics service that allows users to query data using natural language.

The system workflow is:

1. A user uploads a dataset.
2. The system parses and normalizes the data.
3. The data is stored in a database.
4. The user can ask natural language questions.
5. The system interprets the question.
6. The system builds a structured query.
7. The query is converted to SQL.
8. The data is retrieved.
9. The result is returned as tables, charts, or dashboards.

The goal is to create a fully local system capable of natural language analytics similar to modern AI BI tools.

SYSTEM ARCHITECTURE

The system is modular. Each module has a clear responsibility.

Main modules include:

1. FILE INGESTION MODULE

Responsible for:

* accepting uploaded files
* parsing CSV and XLSX
* detecting schema
* normalizing data structure
* storing data in ClickHouse

Technologies typically used:
pandas
pyarrow

---

2. METADATA LAYER

Metadata about datasets is stored in PostgreSQL.

This includes:

* dataset definitions
* schema information
* metrics registry
* dimensions registry
* semantic mappings

---

3. QUERY PARSING MODULE

Natural language queries are parsed using:

nvidia/gliner-pii

This extracts entities such as:

* metrics
* brands
* filters
* limits
* offsets
* sorting
* date expressions

---

4. DATE NORMALIZATION MODULE

Natural language dates are processed in two stages.

Stage 1:
An Ollama LLM converts date phrases into normalized English expressions.

Stage 2:
Duckling converts the normalized phrase into a precise date interval.

Example:

"q1 2024"

becomes

{
"start": "2024-01-01",
"end": "2024-03-31"
}

---

5. SEMANTIC RESOLUTION MODULE

Business terms such as metrics, brands, and sorting keywords are resolved using embeddings.

User phrases are embedded using:

google/embeddinggemma-300m

Embeddings are stored and searched using:

Qdrant vector database

This allows mapping user phrases to internal business concepts.

Example:

"top selling brands"

may map to:

metric = revenue
dimension = brand
sort = revenue desc

---

6. SEMANTIC QUERY OBJECT

User queries are converted into an intermediate structure called a Semantic Query Object (SQO).

Example:

{
"metrics": ["revenue"],
"dimensions": ["brand"],
"filters": [],
"date_range": {
"start": "2024-01-01",
"end": "2024-03-31"
},
"sort": [
{"field":"revenue","direction":"desc"}
],
"limit": 10,
"offset": 0,
"visualization": "bar_chart"
}

This object will later be converted into SQL.

SQL generation must be deterministic and not AI-generated.

---

7. QUERY EXECUTION

Queries are executed against ClickHouse.

ClickHouse stores analytical data and fact tables.

---

8. VISUALIZATION LAYER

Query results are transformed into user-friendly outputs.

Visualization is done using:

Plotly

Possible outputs include:

* tables
* bar charts
* line charts
* dashboards
